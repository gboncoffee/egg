Saved:
+ ~ra~ (Return address)
+ ~sp~ (Stack pointer)
+ ~sa~
+ ~sb~
+ ~sc~
+ ~sd~

Temporary:
+ ~ta~
+ ~tb~
+ ~tc~
+ ~td~
+ ~te~
+ ~tf~
+ ~tg~
+ ~th~

Function arguments are passed via the temporary registers.

* Environment
There are two protection rings: Ring-0 and Ring-1.

Ring-0 processes can call firmware-defined services via the ~fwcall~. Arbitrary interruptions can be raised via the ~int~ instruction. Ring-1 processes can call the operating system via the ~syscall~ and ~sysrcall~ instructions. Ring-1 processes can trigger a debugger break interruption via the ~dbgbrk~ instruction. The same instruction can be used by the operating system to trigger a firmware-defined debugger service.

The firmware may provide up to 256 control registers, only accessible to Ring-0 processes. The operating system can get a firmware identifier via the ~fwid~ instruction, thus being able to get the number of control registers. The number and behaviour of control registers shall not change for the same firmware ID.

The ~syscall~ instructions places the immediate in the ~t0~ register. ~sysrcall~ is the equivalent but not changing the value of ~t0~, thus performing an arbitrary call.

* Addressing
Flat 32 bit space addressing. Firmwares may support virtual memory pagination. Instructions may only be executed when their addresses have the least significant bit 0.
